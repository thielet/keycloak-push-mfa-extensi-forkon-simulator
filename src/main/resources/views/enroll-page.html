Why is the initial Push-MFA enrollment signed?

During the initial enrollment, the client does not yet possess a trusted authentication factor.

Signing the enrollment response ensures that the process cannot be manipulated or replayed by a third party.

It provides cryptographic proof that the enrollment was issued by a trusted backend and is bound to this specific user and device.

To support this, the button on this page generates a JWK key pair and downloads it as a JSON file.

This file must then be placed into the designated key location of the mock system so that enrollment signatures can be validated correctly.
